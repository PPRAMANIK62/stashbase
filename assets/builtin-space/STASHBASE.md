# StashBase Rules — 👋 Start Here (example)

This file is where you tell AI assistants how to look after a space. Drop a
`STASHBASE.md` into any space (or the knowledge-base root) and assistants
read it through the `kb_info` tool before they touch anything, then follow
it as they work — summarizing, filing, and tidying as a side effect of the
work you were already doing.

This space is a read-only tour, so the rules below are just examples —
nothing here actually gets reorganized. In your own spaces, write whatever
keeps the place the way you want it, for example:

- Keep one running summary note per topic; fold new material into it instead
  of letting loose files pile up.
- File meeting notes under `Meetings/`, named `YYYY-MM-DD <topic>`.
- Summarize anything longer than ~2,000 words into a short note up top, and
  leave the original untouched.
- Never edit files under `Archive/`.

It's plain language — there's no schema to learn. Be as loose or as specific
as you like; the assistant reads it and uses its judgment.
